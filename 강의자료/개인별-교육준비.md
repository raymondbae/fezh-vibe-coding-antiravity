## 🌐 로컬 웹 서비스 외부 공개 방법 (실습 공유용)

실습 중에 프론트엔드와 백엔드가 분리된 로컬 웹 서비스를 모바일 기기나 동료의 PC에서 접속할 수 있게 외부로 원활하게 공유하는 가장 깔끔하고 전문적인 방법들을 안내합니다.

### 🌟 [추천] 방법 1. Vite Proxy + Cloudflare Quick Tunnel (가장 우아한 솔루션)
프론트엔드 포트 하나만 터널링하여, API URL 변경이나 CORS(크로스 오리진) 이슈 없이 원격 접속까지 한 번에 끝내는 최고 효율의 구성입니다.

#### **1. 프론트엔드(Vite) Proxy 설정 (단일 포트로 통합)**
프론트엔드 프로젝트의 `vite.config.js`(또는 `vite.config.ts`) 파일에 proxy 설정을 추가하여 프론트엔드 서버가 백엔드 API 서버를 대리 호출하도록 구성합니다.

```javascript
// vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/react'

export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      // 프론트엔드로 들어오는 모든 '/api'로 시작하는 요청을 백엔드(3000포트)로 토스
      '/api': {
        target: 'http://localhost:3000',
        changeOrigin: true,
        secure: false,
      }
    }
  }
})
```

#### **2. 프론트엔드 API 호출 주소 단순화**
코드에서 API를 호출할 때 더 이상 절대 경로(`http://localhost:3000/api/...`)를 쓰지 않고, 단순한 **상대 경로**로 작성합니다.
* **수정 전**: `fetch("http://localhost:3000/api/users")`
* **수정 후**: `fetch("/api/users")`
*(CORS 에러가 원천 차단되며 환경 변수 관리의 번거로움이 완전히 사라집니다.)*

#### **3. Cloudflare Quick Tunnel 실행 (단 한 줄로 외부 공개)**
터미널을 하나 더 열고 프론트엔드 포트(`5173`)만 지정해 Cloudflare 무료 퀵 터널 명령어를 실행합니다. **(가입, 로그인 필요 없음)**
```bash
npx cloudflared tunnel --url http://localhost:5173
```
* **접속 방법**: 명령어가 켜지면 터미널 창에 즉시 생성되는 임시 HTTPS 도메인 주소(예: `https://your-domain.trycloudflare.com`)를 복사해 다른 사람이나 스마트폰 브라우저에 치면 즉시 완벽히 연동된 웹 서비스가 구동됩니다. (LocalTunnel 등과 달리 최초 접속 시 IP를 입력하는 보안 경고 창도 전혀 뜨지 않는 쾌적함을 보여줍니다.)

### 🔹 방법 2. VS Code 내장 포트 포워딩 (가장 직관적인 GUI 방식)
명령어 치는 것도 번거로운 경우, VS Code에 내장된 강력한 GUI 포트 포워딩 기능을 이용할 수도 있습니다.

#### **사용 순서**
1. **[포트] 탭으로 이동**: VS Code 하단 터미널 영역 옆에 있는 **[포트 (Ports)]** 탭을 클릭합니다.
2. **포트 추가**: **[포트 추가 (Add Port)]** 버튼을 눌러 공유하고자 하는 포트 번호(예: `5173`, `3000`)를 각각 기입합니다.
3. **공개 상태 변경 (매우 중요)**: 추가된 포트 항목의 '포트 표시 여부 (Port Visibility)' 항목을 마우스 우클릭하여 **[Public (공개)]**으로 바꿉니다. (Private 상태이면 접속자에게 로그인 창이 뜨게 됩니다.)
4. **주소 공유**: '전달된 주소 (Forwarded Address)'에 자동 발급된 주소를 복사하여 공유하면 즉시 외부 진입이 허용됩니다!
*(단, 프론트엔드 코드 내의 백엔드 API URL 주소를 VS Code가 백엔드용으로 발급해준 Forwarded Address로 직접 변경해 주어야 원활한 API 통신이 가능합니다.)*

## 🚀 완성된 웹 서비스 영구 배포 방법 (Vercel 활용)

로컬 터널링을 넘어 실습 프로젝트 결과물을 24시간 중단 없이 전 세계 누구나 접속할 수 있도록 Vercel 클라우드 플랫폼에 평생 무료(Hobby Plan)로 영구 배포하는 가장 빠른 가이드입니다.

### **[1단계] GitHub 저장소 생성 및 코드 Push**
1. **GitHub 로그인 및 새 저장소 생성**: [GitHub](https://github.com/)에 로그인하여 우측 상단의 `+` 버튼 -> **New repository**를 클릭해 빈 저장소를 만듭니다. (예: `my-vibe-project`)
2. **로컬 코드 푸시**: 터미널에서 작업 중인 디렉토리로 이동하여 아래 명령어를 통해 코드를 GitHub에 업로드합니다:
   ```bash
   git add .
   git commit -m "feat: 최종 결과물 완성"
   git branch -M main
   git remote add origin https://github.com/본인계정이름/my-vibe-project.git
   git push -u origin main
   ```

### **[2단계] Vercel 회원가입 및 GitHub 계정 연동**
1. **Vercel 홈페이지 접속**: [Vercel](https://vercel.com/) 사이트에 진입하여 우측 상단 **[Sign Up]** 버튼을 누릅니다.
2. **GitHub 계정으로 가입**: 가입 수단 중 **[Continue with GitHub]**을 클릭하여 기존의 깃허브 계정으로 간편하게 인증을 마치고 가입을 완료합니다.

### **[3단계] 프로젝트 선택 및 원클릭 배포**
1. **저장소 가져오기**: Vercel 대시보드 메인 화면에서 **[Add New...]** -> **[Project]** 단추를 누릅니다.
2. **Import 실행**: 깃허브 연동 리스트에서 방금 생성한 레포지토리(예: `my-vibe-project`)를 찾아 우측의 **[Import]** 버튼을 클릭합니다.
3. **설정 확인 및 배포**:
   * **Framework Preset**: 사용 중인 스택(예: Vite)이 자동으로 올바르게 선택되었는지 확인합니다.
   * 별도의 빌드 설정을 건드릴 필요 없이 맨 하단의 **[Deploy]** 버튼을 누릅니다.
   * 약 30초~1분 이내에 빌드가 끝나고 폭죽 이펙트와 함께 배포가 성공적으로 완료됩니다!

### **[4단계] 💡 [실전 꿀팁] 환경 변수(Environment Variables) 설정**
프로젝트 내부에서 백엔드 API 서버 주소 등 중요한 보안 키를 사용하는 경우, Vercel 관리자 페이지에서 안전하게 환경 변수를 바인딩할 수 있습니다.
1. Vercel 프로젝트 대시보드에서 **[Settings]** -> **[Environment Variables]** 탭으로 이동합니다.
2. **Key-Value 추가**: 
   * **Key**: `VITE_API_URL` (또는 해당 프로젝트에 필요한 환경변수 명칭)
   * **Value**: 실제 가동 중인 외부 백엔드 API 주소 (예: `https://api.your-service.com`)
3. **[Save]** 버튼을 클릭한 뒤, 상단 **[Deployments]** 탭에서 최근 배포된 빌드 우측의 `...` -> **Redeploy**를 누르면 환경 변수가 완벽히 반영됩니다.

### **[5단계] 영구 무료 고유 URL 배포 확인**
배포 성공 후 Vercel 대시보드 화면에 나오는 `https://내-프로젝트.vercel.app` 형태의 고유 URL 주소를 클릭하여 접속을 테스트합니다. 이제 노트북을 끄거나 잠들어도 24시간 언제나 모바일, 태블릿, 다른 컴퓨터에서 이 주소를 통해 내 결과물을 자랑스럽게 보여줄 수 있습니다!
