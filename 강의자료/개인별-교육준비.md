## 🌐 로컬 웹 서비스 외부 공개 방법 (실습 공유용)

실습 중에 프론트엔드와 백엔드가 분리된 로컬 웹 서비스를 모바일 기기나 동료의 PC에서 접속할 수 있게 외부로 원활하게 공유하는 가장 깔끔하고 전문적인 방법들을 안내합니다.

### 🌟 [추천] 방법 1. Vite Proxy + Cloudflare Quick Tunnel (가장 우아한 솔루션)
프론트엔드 포트 하나만 터널링하여, API URL 변경이나 CORS(크로스 오리진) 이슈 없이 원격 접속까지 한 번에 끝내는 최고 효율의 구성입니다.

#### **1. 프론트엔드(Vite) Proxy 설정 (단일 포트로 통합)**
프론트엔드 프로젝트의 `vite.config.js`(또는 `vite.config.ts`) 파일에 proxy 설정을 추가하여 프론트엔드 서버가 백엔드 API 서버를 대리 호출하도록 구성합니다.

```javascript
// vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/react'

export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      // 프론트엔드로 들어오는 모든 '/api'로 시작하는 요청을 백엔드(3000포트)로 토스
      '/api': {
        target: 'http://localhost:3000',
        changeOrigin: true,
        secure: false,
      }
    }
  }
})
```

#### **2. 프론트엔드 API 호출 주소 단순화**
코드에서 API를 호출할 때 더 이상 절대 경로(`http://localhost:3000/api/...`)를 쓰지 않고, 단순한 **상대 경로**로 작성합니다.
* **수정 전**: `fetch("http://localhost:3000/api/users")`
* **수정 후**: `fetch("/api/users")`
*(CORS 에러가 원천 차단되며 환경 변수 관리의 번거로움이 완전히 사라집니다.)*

#### **3. Cloudflare Quick Tunnel 실행 (단 한 줄로 외부 공개)**
터미널을 하나 더 열고 프론트엔드 포트(`5173`)만 지정해 Cloudflare 무료 퀵 터널 명령어를 실행합니다. **(가입, 로그인 필요 없음)**
```bash
npx cloudflared tunnel --url http://localhost:5173
```
* **접속 방법**: 명령어가 켜지면 터미널 창에 즉시 생성되는 임시 HTTPS 도메인 주소(예: `https://your-domain.trycloudflare.com`)를 복사해 다른 사람이나 스마트폰 브라우저에 치면 즉시 완벽히 연동된 웹 서비스가 구동됩니다. (LocalTunnel 등과 달리 최초 접속 시 IP를 입력하는 보안 경고 창도 전혀 뜨지 않는 쾌적함을 보여줍니다.)

### 🔹 방법 2. VS Code 내장 포트 포워딩 (가장 직관적인 GUI 방식)
명령어 치는 것도 번거로운 경우, VS Code에 내장된 강력한 GUI 포트 포워딩 기능을 이용할 수도 있습니다.

#### **사용 순서**
1. **[포트] 탭으로 이동**: VS Code 하단 터미널 영역 옆에 있는 **[포트 (Ports)]** 탭을 클릭합니다.
2. **포트 추가**: **[포트 추가 (Add Port)]** 버튼을 눌러 공유하고자 하는 포트 번호(예: `5173`, `3000`)를 각각 기입합니다.
3. **공개 상태 변경 (매우 중요)**: 추가된 포트 항목의 '포트 표시 여부 (Port Visibility)' 항목을 마우스 우클릭하여 **[Public (공개)]**으로 바꿉니다. (Private 상태이면 접속자에게 로그인 창이 뜨게 됩니다.)
4. **주소 공유**: '전달된 주소 (Forwarded Address)'에 자동 발급된 주소를 복사하여 공유하면 즉시 외부 진입이 허용됩니다!
*(단, 프론트엔드 코드 내의 백엔드 API URL 주소를 VS Code가 백엔드용으로 발급해준 Forwarded Address로 직접 변경해 주어야 원활한 API 통신이 가능합니다.)*

## 🚀 완성된 웹 서비스 영구 배포 방법 (Vercel 활용)

로컬 터널링을 넘어 실습 프로젝트 결과물을 24시간 중단 없이 전 세계 누구나 접속할 수 있도록 Vercel 클라우드 플랫폼에 평생 무료(Hobby Plan)로 영구 배포하는 가장 빠른 가이드입니다.

### **[1단계] GitHub 저장소 생성 및 코드 Push**
1. **GitHub 로그인 및 새 저장소 생성**: [GitHub](https://github.com/)에 로그인하여 우측 상단의 `+` 버튼 -> **New repository**를 클릭해 빈 저장소를 만듭니다. (예: `my-vibe-project`)
2. **로컬 코드 푸시**: 터미널에서 작업 중인 디렉토리로 이동하여 아래 명령어를 통해 코드를 GitHub에 업로드합니다:
   ```bash
   git add .
   git commit -m "feat: 최종 결과물 완성"
   git branch -M main
   git remote add origin https://github.com/본인계정이름/my-vibe-project.git
   git push -u origin main
   ```

### **[2단계] Vercel 회원가입 및 GitHub 계정 연동**
1. **Vercel 홈페이지 접속**: [Vercel](https://vercel.com/) 사이트에 진입하여 우측 상단 **[Sign Up]** 버튼을 누릅니다.
2. **GitHub 계정으로 가입**: 가입 수단 중 **[Continue with GitHub]**을 클릭하여 기존의 깃허브 계정으로 간편하게 인증을 마치고 가입을 완료합니다.

### **[3단계] 프로젝트 선택 및 원클릭 배포**
1. **저장소 가져오기**: Vercel 대시보드 메인 화면에서 **[Add New...]** -> **[Project]** 단추를 누릅니다.
2. **Import 실행**: 깃허브 연동 리스트에서 방금 생성한 레포지토리(예: `my-vibe-project`)를 찾아 우측의 **[Import]** 버튼을 클릭합니다.
3. **설정 확인 및 배포**:
   * **Framework Preset**: 사용 중인 스택(예: Vite)이 자동으로 올바르게 선택되었는지 확인합니다.
   * 별도의 빌드 설정을 건드릴 필요 없이 맨 하단의 **[Deploy]** 버튼을 누릅니다.
   * 약 30초~1분 이내에 빌드가 끝나고 폭죽 이펙트와 함께 배포가 성공적으로 완료됩니다!

### **[4단계] 💡 [실전 꿀팁] 환경 변수(Environment Variables) 설정**
프로젝트 내부에서 백엔드 API 서버 주소 등 중요한 보안 키를 사용하는 경우, Vercel 관리자 페이지에서 안전하게 환경 변수를 바인딩할 수 있습니다.
1. Vercel 프로젝트 대시보드에서 **[Settings]** -> **[Environment Variables]** 탭으로 이동합니다.
2. **Key-Value 추가**: 
   * **Key**: `VITE_API_URL` (또는 해당 프로젝트에 필요한 환경변수 명칭)
   * **Value**: 실제 가동 중인 외부 백엔드 API 주소 (예: `https://api.your-service.com`)
3. **[Save]** 버튼을 클릭한 뒤, 상단 **[Deployments]** 탭에서 최근 배포된 빌드 우측의 `...` -> **Redeploy**를 누르면 환경 변수가 완벽히 반영됩니다.

### **[5단계] 영구 무료 고유 URL 배포 확인**
배포 성공 후 Vercel 대시보드 화면에 나오는 `https://내-프로젝트.vercel.app` 형태의 고유 URL 주소를 클릭하여 접속을 테스트합니다. 이제 노트북을 끄거나 잠들어도 24시간 언제나 모바일, 태블릿, 다른 컴퓨터에서 이 주소를 통해 내 결과물을 자랑스럽게 보여줄 수 있습니다!

## 💡 부록: 프론트엔드 + Node.js 백엔드 API 통합 배포 & 로컬 테스트 가이드

프로젝트가 커져서 프론트엔드뿐만 아니라 자체 백엔드 API 서버(Node.js)까지 구축해야 하는 경우, 단 하나의 GitHub 저장소에 두 코드를 모두 밀어 넣고 무료로 동시 배포 및 로컬 테스트를 진행할 수 있는 강력한 방법입니다.

### **1. 📂 모노레포(Monorepo) 폴더 구조 설계**
Vercel은 프로젝트 최상위 루트의 **`/api`** 폴더 내에 있는 JavaScript/TypeScript 파일들을 자동으로 클라우드 백엔드 API(Serverless Functions)로 호스팅해 줍니다.

```text
my-vibe-project/
├── index.html         # 프론트엔드 마크업 (Vite)
├── src/               # React / Vue 등 프론트엔드 소스 코드
├── vite.config.js
└── api/               # ⭐️ 백엔드 Node.js API 폴더 (Vercel 치트키)
    ├── hello.js       # -> 접속 주소: https://내-도메인/api/hello
    └── users.js       # -> 접속 주소: https://내-도메인/api/users
```

### **2. 📄 Node.js API 코드 샘플 (`api/users.js`)**
별도로 Express 패키지를 설치하거나 `app.listen(3000)` 같은 무거운 리스너를 열 필요가 없습니다. Express의 Request, Response 객체와 100% 호환되는 간결한 핸들러 함수 하나만 `default export`해 주면 됩니다.

```javascript
// api/users.js - Vercel 전용 Node.js 백엔드 서버리스 핸들러 함수
export default function handler(request, response) {
  // CORS 에러 방지를 위한 공통 헤더 추가 (동일 도메인 호출 시에는 생략 가능)
  response.setHeader('Access-Control-Allow-Origin', '*');
  
  // 데이터베이스 조회 대용 샘플 리스트 데이터 생성
  const usersList = [
    { id: 1, name: '이순신', role: '리드 개발자' },
    { id: 2, name: '홍길동', role: 'UI/UX 디자이너' },
    { id: 3, name: '임꺽정', role: '프로젝트 매니저' }
  ];
  
  // HTTP 상태코드 200번(성공) 및 JSON 데이터 응답 처리
  return response.status(200).json({
    success: true,
    data: usersList
  });
}
```

### **3. 💻 로컬 컴퓨터에서 실시간 테스트 및 디버깅 방법 (`npx vercel dev`)**
Vercel에 올리기 전, 내 컴퓨터 로컬 환경에서 프론트엔드와 백엔드 API를 묶어서 실시간으로 테스트하고 핫 리로딩(Hot-Reloading)을 구현하는 가장 똑똑한 방법은 **Vercel CLI 에뮬레이터**를 사용하는 것입니다.

1. **에뮬레이터 구동**: 터미널을 열고 프로젝트 루트 경로에서 아래 명령어 한 줄을 실행합니다. (글로벌 설치 불필요)
   ```bash
   npx vercel dev
   ```
2. **최초 연동 설정 (딱 1회만 엔터 치며 연동)**:
   * `Set up and deploy ...?` ➡️ **`y`** 입력
   * `Which scope ...?` ➡️ **엔터** (내 계정 자동 선택)
   * `Link to existing project?` ➡️ **`n`** 입력 (새로운 Vercel 프로젝트 생성)
   * `What's your project's name?` ➡️ **엔터** (폴더 이름 그대로 사용)
   * `In which directory ...?` ➡️ **엔터** (현재 루트 경로 사용)
3. **통합 로컬 주소 접속**: 설정이 완료되면 Vercel CLI가 내 컴퓨터에 로컬 가상 구동 환경을 열어줍니다.
   * 로컬 구동 주소: **`http://localhost:3000`**
   * **장점**: 이 단 하나의 포트로 접속하면, 리액트 화면(Vite)이 실시간으로 고치는 대로 즉시 갱신될 뿐만 아니라, 프론트에서 호출하는 `/api/users` 경로도 백엔드 Node.js 파일(`api/users.js`)을 즉시 호출하여 완벽하게 데이터 통신을 가상 에뮬레이션해 줍니다!

### **4. 🔑 [로컬 테스트용 꿀팁] 로컬 환경 변수(`.env.local`) 연동**
Vercel 로컬 서버는 최상위의 `.env.local` 파일도 자동으로 감지하여 에뮬레이터 구동 시 백엔드 코드의 `process.env.변수명`에 매핑해 줍니다.
* **설정**: 루트 디렉토리에 `.env.local` 생성 ➡️ `MY_SECRET_KEY=local_key_value_123` 입력
* **소스 내 사용**: `api/users.js` 파일 내부에서 `const key = process.env.MY_SECRET_KEY;`로 바로 호출하여 로컬 비밀번호 연동 디버깅 가능!
* **장점**: 깃허브에는 `.env.local`이 업로드되지 않아(보안 유지) 안전하며, 실 서버 배포 시에는 Vercel 클라우드 설정창의 환경 변수를 알아서 가리키므로 코드 변경이 아예 불필요합니다.

## 💡 부록 2: 전통적인 Express 앱 통째로 Vercel 서버리스로 호스팅하는 이중화 비법

Vercel의 개별 파일형 API 대신, 기존에 완벽히 구동되던 Express 어플리케이션 전체를 하나의 파일로 Vercel 서버리스 환경 위에 얹어 배포하는 치트키입니다. 소스코드 변경 없이 로컬 PC와 Vercel 양쪽에서 모두 완벽히 돌아가도록 스마트하게 설계하는 이중화 기법입니다.

### **1. 📂 프로젝트 및 라우터 매핑 (`vercel.json`)**
프로젝트 루트 디렉토리에 **`vercel.json`** 파일을 만들고 아래 코드를 작성합니다. 이 설정은 `/api`로 들어오는 모든 경로 요청을 `api/index.js` 단 하나의 파일로 쏘아 주는 교통정리 역할을 합니다.

```json
{
  "rewrites": [
    {
      "source": "/api/(.*)",
      "destination": "/api/index.js"
    }
  ]
}
```

### **2. 📄 소스코드 무수정(Zero-Modification) 이중화 소스 (`api/index.js`)**
Vercel 클라우드 서버에 빌드가 완료되어 실행될 때는 자동으로 `process.env.NODE_ENV = 'production'` 환경 변수가 활성화됩니다. 이를 이용해 로컬 서버 구동과 서버리스 호스팅을 우아하게 자동 분기 처리합니다.

```javascript
// api/index.js - 전통적인 Express 코드를 그대로 작성!
const express = require('express');
const app = express();

app.use(express.json());

// 1. 일반적인 Express 라우팅 그대로 선언
app.get('/api/hello', (req, res) => {
  res.json({ message: "익스프레스 백엔드 연동 성공!" });
});

app.get('/api/users', (req, res) => {
  res.json({
    success: true,
    users: [
      { id: 1, name: "이순신", role: "리드 개발자" },
      { id: 2, name: "강감찬", role: "UI 디자이너" }
    ]
  });
});

// 2. ⭐️ [이중화 치트키] 로컬 구동 vs Vercel 서버리스 자동 분기 로직
if (process.env.NODE_ENV !== 'production') {
  // Vercel 클라우드가 아닐 때(즉, 내 로컬 컴퓨터 터미널에서 구동할 때만) 포트를 열어 리스닝을 시작합니다.
  const PORT = process.env.PORT || 3000;
  app.listen(PORT, () => {
    console.log(`[LOCAL] 익스프레스 서버가 http://localhost:${PORT} 에서 구동 중입니다.`);
  });
}

// Vercel이 이 파일을 통째로 읽어 들여 서버리스 함수로 호출할 수 있도록 app 자체를 내보냅니다.
module.exports = app;
```

## 💡 부록 3: SQLite를 클라우드 무료 배포 환경에서 온전히 쓰기 위한 최고의 솔루션

Vercel과 같은 서버리스 환경은 파일 시스템이 **휘발성(Ephemeral)**이기 때문에 로컬에 적는 SQLite 파일(`*.sqlite`) 데이터가 몇 분 마다 계속 완전히 포맷되어 유실되는 명확한 기술적 한계가 존재합니다. 이를 완벽하게 우회하여 클라우드 환경에서도 SQLite의 편리함을 완전 무료로 누릴 수 있는 대세 솔루션 2가지를 전수합니다.

### **1. 🚀 대세 중의 대세: 서버리스 클라우드 SQLite "Turso (터소)"**
Turso는 로컬의 SQLite 문법과 가벼움을 그대로 유지하면서 데이터베이스 파일 자체는 클라우드 망에 저장해 주는 **분산 서버리스 SQLite 서비스**입니다.

* **왜 강력한가요?**:
  * 100% 동일한 SQL 표준 쿼리 문법(SQLite 호환 libSQL 엔진) 사용.
  * **평생 무료 플랜(Starter)**에서 데이터베이스를 최대 **500개**, 총 **9 GB** 용량까지 아낌없이 무료로 줍니다.
* **Vercel 백엔드 코드 연동 예시**:
  ```javascript
  // SQLite 라이브러리 대신 Turso(libSQL) 클라이언트 라이브러리 설치 및 연동
  const { createClient } = require("@libsql/client");
  
  const db = createClient({
    url: "libsql://내-터소-고유식별주소.turso.io", // Turso 클라우드 주소
    authToken: "내-보안-토큰"                     // Vercel 환경 변수로 바인딩
  });
  
  app.get('/api/users', async (req, res) => {
    // SQLite와 완벽히 호환되는 쿼리 실행
    const result = await db.execute("SELECT * FROM users");
    res.json({ success: true, data: result.rows });
  });
  ```

### **2. 📦 영구 볼륨 디스크(Volume)를 무료로 주는 호스팅 "Fly.io"**
컨테이너 기반 서버를 배포할 때, 리셋되지 않고 데이터가 영구 보존되는 **3GB 무료 외장 하드디스크 볼륨**을 끼워 주는 차세대 호스팅 클라우드입니다.

* **어떻게 구동하나요?**:
  1. Fly.io에 Express 서버를 배포할 때 무료 디스크 볼륨을 생성하고 서버 컨테이너에 마운트(예: `/data` 폴더)합니다.
  2. 내 백엔드 코드 내의 SQLite 데이터 파일의 경로를 해당 마운트 폴더 내 주소(예: `/data/database.sqlite`)로 바라보게 세팅합니다.
  3. **결과**: 서버 컨테이너가 아무리 꺼졌다 켜지고 재빌드 되어도 외장 하드디스크 내부의 SQLite 데이터는 손상 없이 영구적으로 안전하게 살아남습니다!
